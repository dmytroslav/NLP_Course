# NER Pipeline Notes (Lab 10)

**1. Який pipeline вибрано**
Для виконання роботи обрано spaCy з офіційною українською моделлю `uk_core_news_sm`.

**2. Які сутності він уміє з коробки**
Стандартна модель розпізнає базові сутності: `PER` (персони), `LOC` (локації), `ORG` (організації).

**3. Які доменні сутності для вашого корпусу важливі**
Оскільки корпус складається з новин Telegram військової тематики, критично важливими є: військові абревіатури (ЗСУ, ППО), типи озброєння (`WEAPON`: Storm Shadow, HIMARS), специфічні державні установи, а також дати (включаючи час) та грошові суми (`MONEY`).

**4. Які правила ви додали**
Гібридний шар складається з двох компонентів:
* `EntityRuler`: словник для жорсткого пошуку військових та міжнародних організацій (Генштаб, ООН, НАТО) та типів зброї.
* `Regex`: кастомний компонент для вилову дат (напр., "Сьогодні вночі", "Вчора о 15:00") та грошових сум ("400 млн доларів").

**5. Що стало краще після правил**
Модель перестала ігнорувати ключові доменні сутності (`WEAPON`, специфічні `ORG`). Суттєво покращилося розпізнавання нестандартних форматів часу та грошових сум, які бейзлайн повністю пропускав.

**6. Які помилки залишилися**
* **False positives:** Широкі патерни регулярних виразів іноді захоплюють зайве (наприклад, "15 ворожих" розпізнається як `DATE`).
* **Tokenization issues:** Проблеми з абревіатурами, що містять апострофи (Держспецзв'язку).
* **Boundary errors:** Модель усе ще захоплює лише частину довгих складених назв (наприклад, "Національний банк *України*").

**7. Що б ви фіксили далі**
Потрібно звузити регулярні вирази для дат, щоб уникнути помилкових спрацьовувань, розширити словник `EntityRuler` більшою кількістю військових підрозділів та додати postprocessing правило для об'єднання (merge split spans) складених назв організацій.
