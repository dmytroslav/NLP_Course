# Dataset Card: Ukrainian Fake News Classification

## Project
Класифікація українських новин на правдиві та фейкові (пропагандистські).

## Task
[cite_start]**Type:** A. Text Classification (Binary) [cite: 6]
**Input:** Текст заголовка новини (Headline).
**Output:** Мітка класу (True / Fake).

## Data Source
**Original Source:** Kaggle "Ukrainian fake and true news" (Dataset 4).
**URL:** https://www.kaggle.com/datasets/zepopo/ukrainian-fake-and-true-news/data
**Origin:** Дані зібрані з Telegram-каналів (Суспільне, Perepichka, та проросійські фейкові канали).

## [cite_start]Volume [cite: 62]
- **Кількість текстів:** ~10,700
- **Кількість класів:** 2 (True, Fake)
- **Розподіл:** ~8237 (True) vs ~2498 (Fake).

## [cite_start]Language & Domain [cite: 63]
- **Мова:** Українська (UA).
- **Домен:** Новини воєнного часу, політика, соціум (2022 рік).

## [cite_start]Cleaning Process [cite: 64]
1. Видалення зайвих пробілів та переносів рядків.
2. Уніфікація апострофів (всі варіанти зведені до `'`).
3. Заміна посилань на токен `<URL>` та email-адрес на `<EMAIL>`.
4. Видалення (або маркування) точних дублікатів.

## [cite_start]Risks [cite: 65]
1. **Дисбаланс класів:** Правдивих новин у 3 рази більше, ніж фейкових. Модель може схилятися до передбачення мажоритарного класу.
2. **Зміщення джерела (Source Bias):** Оскільки фейки бралися зі специфічних каналів, модель може вивчити стиль цих каналів, а не саму суть неправдивості.
3. **Шум:** Наявність дуже коротких заголовків (менше 5 слів).

## Updates for Lab 4 (Information Extraction)
- **Витягнуті поля:** DATE (дати), AMOUNT (грошові суми), LOCATION (міста/регіони).
- **Ризики / PII:** Витягування LOCATION може опосередковано розкривати геолокацію певних подій, але для новин це публічна інформація. Грошові суми (AMOUNT) не прив'язані до конкретних осіб (PII). Раніше замасковані `<URL>`, `<EMAIL>`, `<PHONE>` залишаються анонімізованими.



## Splits & Leakage (Lab 5)
- **Розбиття**: Stratified random split (80/10/10) по колонці `label`.
- **Баланс**: Співвідношення класів 76.7% / 23.3% ідеально збережено у всіх вибірках.
- **Виявлені ризики**: Знайдено незначний leakage (22 точні дублікати між train та test, 23 між train та val), які потребують очищення перед навчанням.


## Аналіз особливостей даних (за результатами ЛР7)
- **Дисбаланс класів (Imbalance):** Наявний (76.7% True / 23.3% False). Використання зважених класів є обов'язковим для уникнення деградації міноритарного класу.
- **Шум у тексті (Noisy text):** Дані містять значну кількість нестандартної лексики, трансліту та специфічного форматування (особливо з Telegram). Використання символьних n-грам (char-ngrams) довело свою ефективність, значно покращивши загальні метрики.
- **Найпроблемніший клас:** False (Фейки), оскільки вони часто мімікрують під структуру офіційних повідомлень (клас True).
- **Threshold логіка:** Дефолтний поріг рішення балансує результати. Однак для продуктового рішення (наприклад, фактчекінг-бота) доцільним є компроміс у бік вищого Recall для класу False (зсув порогу), щоб мінімізувати кількість пропущених маніпуляцій.
