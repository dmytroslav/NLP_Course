# Extraction Schema (Lab 11)

**1. Яка extraction-задача:** Витягнення структурованої інформації про військові інциденти з текстів новин.
**2. Які поля у JSON:** `event_type` (enum), `location` (string/null), `weapons` (array), `casualties` (string/null), `date_text` (string/null).
**3. Які поля required:** `event_type`, `location`, `weapons`.
**4. Як виглядає JSON schema:** (Описана в `src/json_schema.py`). Використовується суворий `enum` для типів подій та масив рядків для зброї. `additionalProperties` встановлено в `False`.
**5. Які правила для null / missing values:** Для відсутніх даних у полях `location`, `casualties` та `date_text` дозволено тип `null`. Якщо зброї не знайдено, має повертатися порожній масив `[]`.
**6. Які поля найчастіше проблемні:** Найпроблемнішим виявилося поле `event_type` — коли текст не релевантний, модель намагається вигадати значення поза межами `enum` (наприклад, "question" або "some_value"). Також масово пропускається required поле `weapons` на коротких/порожніх текстах.
**7. Що repair loop реально виправляє:** Виправляє дрібні синтаксичні порушення схеми (наприклад, коли модель повертає порожній рядок замість валідного enum), змушуючи модель переглянути вивід.
