# Audit Summary (Lab 4)

## Результати Information Extraction
- **DATE precision**: 0.00 (Формат DD.MM.YYYY рідко зустрічається в текстах новин).
- **AMOUNT precision**: ~0.89
- **LOCATION precision**: ~0.84

## Знайдені проблеми (Error Analysis)
1. **LOCATION** іноді витягує міста, які є частиною назви маршруту чи траси (наприклад, "траса Лисичанськ - Бахмут").
2. **AMOUNT** може хибно розпізнавати тарифи (наприклад, "50 грн/л") як виділені суми.
3. **DATE** повністю пропускає дати, написані текстом ("24 лютого").

## Наступні кроки (Анти-правила)
- Застосовано Negative Lookahead `(?!\s*/\s*[а-яa-z]+)` для відсікання тарифів у AMOUNT.
- Додано перевірку лівого контексту (слова "трас", "маршрут", "потяг") для LOCATION.
- Планується підключення словника `months_ua.txt` для розширення пошуку дат.
