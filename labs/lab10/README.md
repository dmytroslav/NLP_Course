# Lab 10: NER Pipeline & Hybrid Rules

**1. Який корпус / evaluation set використано**
Використано очищений корпус `processed_v2.csv`. Створено малий ручний evaluation set на 20 речень, які містять потенційні `LOC`, `ORG`, `PER`, `MONEY`, `DATE` та `WEAPON`.

**2. Який pipeline запущено**
`spaCy` з моделлю `uk_core_news_sm`.

**3. Які rules додано**
* Словник (`EntityRuler`) для специфічних військових абревіатур (ППО, ЗСУ) та зброї (HIMARS, Storm Shadow).
* Регулярні вирази (`Regex NER Component`) для дат, часу та грошових сум.

**4. Що baseline знаходив добре**
Класичні та відомі географічні назви (Бахмут, Херсон, США).

**5. Що baseline пропускав**
Специфічні доменні сутності (ППО, Генштаб), гроші (400 млн доларів) та складні формати дат.

**6. Що rules покращили**
Правила успішно закрили gap у розпізнаванні `WEAPON`, доменних `ORG`, `DATE` та `MONEY`, що суттєво підвищило coverage пайплайну на нашому датасеті.

**7. Які проблеми ще не вирішені**
Залишаються boundary errors (неповне захоплення складених назв) та tokenization issues (розбиття токенів через апострофи).
