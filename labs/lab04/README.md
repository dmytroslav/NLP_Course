# Лабораторна робота №4: Rule-based Information Extraction

## 1. Які 3 типи полів ви витягуєте
- **AMOUNT** (Грошові суми з валютою)
- **LOCATION** (Міста та регіони України)
- **DATE** (Точні дати у форматі DD.MM.YYYY)

## 2. Які правила/словники використали
- Використано регулярні вирази (Regex) для пошуку дат та числових значень.
- Використано словник валют (`currencies.json`) для нормалізації значень (до UAH, USD, EUR).
- Використано закритий словник міст (`locations_ua.txt`) для точного пошуку локацій.
- Застосовано Negative Lookahead (анти-правила) для відсікання тарифів (грн/л) та маршрутів (Потяг Київ-Львів).

## 3. Precision по кожному типу поля (на sample)
| Field Type | Precision |
|------------|-----------|
| AMOUNT     | 0.89      |
| LOCATION   | 0.84      |
| DATE       | 0.00* |
*\* Формат DD.MM.YYYY рідко зустрічається в текстах новин порівняно з текстовим написанням місяців.*

## 4. Топ-5 edge cases у корпусі
1. **Тарифи vs Суми:** `50 грн/л` хибно розпізнавалося як AMOUNT. Вирішено через negative lookahead.
2. **Відсотки:** `100%` могло сприйматися як сума. Вирішено вимогою обов'язкового токена валюти.
3. **Маршрути vs Локації:** `Потяг Київ - Львів` розпізнавав міста як локації подій. Вирішено перевіркою лівого контексту.
4. **Назви установ:** `ТЦ Кримський` захоплювало `Крим`. Вирішено використанням меж слова `\b`.
5. **Текстові дати:** `24 лютого` не розпізнається поточним регулярним виразом.

## 5. Що плануєте покращити в наступних лабах
- Додати словник `months_ua.txt` для розпізнавання дат, записаних текстом.
- Інтегрувати NER-модель (наприклад, spaCy) для більш точного розпізнавання локацій та відсікання власних назв (річок, ТЦ), які збігаються з містами.
