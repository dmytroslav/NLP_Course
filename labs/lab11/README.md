# Lab 11: LLM Extraction (Schema-First)

1. **Extraction-кейс:** Витягнення сутностей військових подій (тип події, локація, зброя, жертви).
2. **Schema:** Жорсткий JSON з `enum` для типів подій, обов'язковими полями та забороною додаткових властивостей (`additionalProperties: false`).
3. **Baseline prompt:** Інструкція безпосередньо вказує формати полів і суворо забороняє будь-який вивід, окрім JSON.
4. **Validator:** Використано бібліотеку `jsonschema` для перевірки типів даних та `enum`.
5. **Repair loop:** Якщо `validate()` падає, модель отримує повідомлення з текстом помилки і має 2 спроби на виправлення власного JSON.
6. **Метрики:** Raw valid JSON rate: **66.7%**. Post-repair valid JSON rate: **73.3%**.
7. **Проблеми, що залишилися:** На текстах, де фізично немає даних для обов'язкових полів, модель намагається вигадати значення поза схемою, що призводить до неможливості відновлення навіть через repair loop.
