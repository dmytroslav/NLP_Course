# Лабораторна робота 6: TF-IDF + Logistic Regression Baseline

## 1. Напрям роботи
Напрям: **A** (Класифікація текстів / Fake News Detection).

## 2. Підзадача
Бінарна класифікація текстів на основі ознаки `label` (True / False). Мета: побудувати базову ML-модель для виявлення патернів у текстах і встановлення метрик-орієнтирів (baselines).

## 3. Baseline-варіанти для порівняння
* **Baseline 1:** `TfidfVectorizer(ngram_range=(1,1))` + `LogisticRegression(class_weight=None)`. Базовий підхід на уніграмах без врахування дисбалансу.
* **Baseline 2:** `TfidfVectorizer(ngram_range=(1,2))` + `LogisticRegression(class_weight='balanced')`. Використання біграм та балансування ваг класів для боротьби з дисбалансом (76.7% vs 23.3%), виявленим у ЛР5.

## 4. Основні цифри (на Test Set)
* **Baseline 1:** Accuracy = 0.8911, Macro-F1 = 0.8185
* **Baseline 2:** Accuracy = 0.8939, Macro-F1 = 0.8510
* **Краща модель:** Baseline 2 (за рахунок параметра `class_weight='balanced'`, який компенсував дисбаланс класів та підняв Macro-F1).

## 5. Error Analysis
Ручний розбір помилок показав, що модель найчастіше помиляється через **перетин лексики (overlap)**: фейки використовують офіційні військові терміни, а реальні новини часто цитують абсурдні заяви ворога, містячи слова-тригери на кшталт "спецоперація". У майбутньому це планується фіксити переходом від TF-IDF (який ігнорує контекст) до семантичних ембеддінгів, а також експериментами з лематизацією.
