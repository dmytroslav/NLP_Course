Аналізувався очищений корпус українських новин та повідомлень Telegram (processed_v2.csv). Були запущені дві тематичні моделі: LSA з використанням TF-IDF та LDA з CountVectorizer. Для кожної моделі перевірялися значення кількості тем k=5 та k=8. Найкращими виявилися теми, виділені моделлю LSA при k=8, зокрема чіткі сюжети про ракетні удари та оголошення повітряних тривог. Поганими вийшли теми, що повністю складалися зі стоп-слів (прийменників) або змішували непов'язані новини через загальну журналістську лексику. Порівняння показало, що LSA дала значно читабельніші та конкретніші результати, тоді як LDA сильно постраждала від технічного шуму та частих слів. Загалом тематичне моделювання виявилося корисним для виявлення великих макро-сюжетів у датасеті, але для кластеризації воно вимагає агресивнішого попереднього очищення корпусу.
